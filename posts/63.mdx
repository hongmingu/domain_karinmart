이번엔 Zsh의 명령어와 alias를 설정하고 스크립트를 작성해본다.

## 명령어 별칭(alias) 설정하기
alias를 사용하면 긴 명령어를 짧은 키워드로 줄여 사용할 수 있다. 
이는 반복적으로 사용하는 명령어의 실행 속도를 높이는 데 효과적이다.

### 별칭 사용법
```
alias ll="ls -l"
alias la="ls -a"
alias lla="ls -la"
```
위의 설정을 ~/.zshrc 파일에 추가하면 ll, la, lla와 같은 별칭으로 ls 명령어를 사용할 수 있다.

## 배열 및 연관 배열 사용하기
zsh에서는 일반 배열과 연관 배열(해시맵과 유사)을 사용할 수 있다.

### 일반 배열
```
fruits=("apple" "banana" "cherry")
echo $fruits[1]  # apple 출력
```
### 연관 배열
```
declare -A person
person=(["name"]="John" ["age"]=30)
echo $person[name]  # John 출력
```
## 글로브 패턴과 확장
글로브 패턴을 사용하면 파일과 디렉토리를 쉽게 찾을 수 있다. zsh는 글로브 패턴에서 다양한 확장 기능을 제공한다.

### 파일 출력하는 패턴
```
* : 모든 파일
**/* : 현재 디렉토리와 모든 하위 디렉토리의 파일
*.txt : 모든 txt 파일
```
### 실제 사용예
```
echo *.txt  # 현재 디렉토리의 모든 .txt 파일을 나열
```
## 스크립트 작성 및 자동화
zsh 스크립트는 명령어를 여러 개 묶어서 자동화할 수 있다.


### backup.sh 스크립트
#!/bin/zsh
```

echo "Starting backup..."
cp /path/to/source /path/to/destination
echo "Backup completed!"
```
##### 스크립트를 실행하기 위해 실행 권한을 부여
```
chmod +x backup.sh
```
##### 그리고 다음과 같이 스크립트를 실행
```
./backup.sh
```
이렇게 zsh 스크립트를 작성하여 일련의 명령어를 자동화할 수 있다.


## 마무리
zsh는 다양한 기능과 확장성을 제공하여 터미널 작업의 생산성을 크게 향상시킬 수 있다. 물론 다른 쉘스크립트도 이런 기능들을 제공하는 경우가 많지만 zsh가 여러가지로 세련된 형태로 제공해주는 것 같다.
